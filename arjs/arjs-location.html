# 基于位置

基于位置仅针对A框架框架实施。

本文使您对基于AR.js的位置有了第一眼的了解。它可以用于室内（但精度较低）和室外AR含量的地理定位。

您可以从HTML或Javascript静态加载位置，也可以从本地/远程json甚至通过API调用加载数据。选择是你的。在上面的文章中，作为教程对所有选项进行了说明。

接下来是API参考。

# gps-camera

必填：是

每个场景允许的最大数量：1

此组件启用位置AR。必须将其添加到camera实体。它可以处理摄像头的位置和旋转，并用于确定用户将设备指向何处。

例如：

<a-camera gps-camera rotation-reader></a-camera>

除此之外，如您在上面的示例中看到的，我们还必须添加rotation-reader以处理旋转事件。有关更多详细信息，请参见此处。

物产

属性 描述 默认值
alert GPS讯号不足时是否显示讯息 positionMinAccuracy 假
positionMinAccuracy 位置信号允许的最低精度 100
minDistance 如果设置，则不会显示与用户的距离小于此值的位置。只允许一个正值。价值以米为单位。 0（禁用）
simulateLatitude 设置此项可以模拟摄像机的纬度，以帮助进行测试。 0（禁用）

simulateAltitude 设置此项可以模拟摄像机海拔高度（以米为单位），以辅助测试。 0（禁用）

# gps-entity-place

必填：是

每个场景允许的最大数量：无限制

此组件使每个实体都可以进行GPS跟踪。这会将特定的世界位置分配给实体，以便用户在其设备指向现实世界中的位置时可以看到它。如果用户距离实体较远，它将显得更小。如果距离太远，将根本看不到它。

它需要纬度和经度作为单个字符串参数（例如a-boxaframe原语）：

<a-box color="yellow" gps-entity-place="latitude: <your-latitude>; longitude: <your-longitude>"/>

另外，您可以使用a帧的“位置”参数来分配y值以更改内容的高度。该值应以当前摄像机高度以上或以下（如果为负）的米数输入。例如，这将指定30米的高度，并将相对于gps相机的当前高度显示：

<a-box color="yellow" gps-entity-place="latitude: <your-latitude>; longitude: <your-longitude>" position="0 30 0"/>

# 特性

如上所示，除了一起定义纬度和经度的字符串外，没有任何其他属性。

# 自定义属性

例如，以下是可以从gps-entity-place实体检索的自定义属性：

const distanceMsg = document.querySelector('[gps-entity-place]').getAttribute('distanceMsg'); console.log(distanceMsg); // "890 meters"

自定义属性 描述 默认值
distance 与用户的距离，在每次用户位置更新时更新。以米为单位的价值。 0
distanceMsg 与用户的距离（以字符串为单位），在每次用户位置更新时都会更新。价值为 meters/kilometers。 ''
