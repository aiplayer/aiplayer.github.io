# 核心API-实体

A-Frame通过元素表示一个实体。正如在实体组件系统模式中定义的那样，实体是占位符对象，我们插入组件为它们提供外观、行为和功能。

在A-Frame中，实体固有地与位置、旋转和缩放组件相连。

# 例子

考虑下面的实体。它本身没有外观、行为或功能。它什么也不做：

<a-entity>

我们可以将组件附加到它以使它呈现或执行某些操作。为了使其具有形状和外观，我们可以附加几何和材质组件：

<a-entity geometry="primitive: box" material="color: red">

或者为了使其发光，我们可以进一步附加灯光组件：

<a-entity geometry="primitive: box" material="color: red"
          light="type: point; intensity: 2.0">

# 检索实体

我们可以使用domapi简单地检索实体。

<a-entity id="mario"></a-entity>

var el = document.querySelector('#mario');

# 属性

# 组件

。组件是附加到该实体的组件的对象。这使我们能够访问实体的组件，包括每个组件的数据、状态和方法。

例如，如果要获取实体的3.js相机对象或材质对象，可以访问其组件：

var camera = document.querySelector('a-entity[camera]').components.camera.camera;
var material = document.querySelector('a-entity[material]').components.material.material;

或者，如果一个组件公开了一个API，我们可以调用它的方法：

document.querySelector('a-entity[sound]').components.sound.pause();

# hasLoaded

实体是否已附加并初始化其所有组件。尽管确保代码在实体准备好之后运行的最佳方法是将代码放在组件中。

# isPlaying

实体是否处于活动状态并正在播放。如果我们暂停实体，那么isPlaying将变为false。

# object3D

.object3D是对实体的3.js object3D表示的引用。更具体地说，object3D是一个3.Group对象，可以包含不同类型的3.object3D，例如相机、网格、灯光或声音： 
// Gaining access to the internal three.js scene graph.
var groupObject3D = document.querySelector('a-entity').object3D;
console.log(groupObject3D.parent);
console.log(groupObject3D.children);

我们可以通过object3DMap访问不同类型的Object3Ds。

# object3DMap

实体的object3DMap是一个对象，它允许访问组件设置的不同类型的3.Object3Ds（例如相机、网格、灯光、声音）。

对于附着了几何图形和灯光组件的实体，object3DMap可能如下所示：

{
  light: <THREE.Light Object>,
  mesh: <THREE.Mesh Object>
}

我们可以使用setObject3D和removeObject3D来管理实体的3.Object3Ds集。

# sceneEl

实体具有对其场景元素的引用。

var sceneEl = document.querySelector('a-scene');
var entity = sceneEl.querySelector('a-entity');
console.log(entity.sceneEl === sceneEl);  // >> true.

# 方法

# addState(stateName)

addState将状态推送到实体上。这将发出stateadded事件，我们可以使用检查状态can是否存在。is:

entity.addEventListener('stateadded', function (evt) {
  if (evt.detail === 'selected') {
    console.log('Entity now selected!');
  }
});

entity.addState('selected');
entity.is('selected');  // >> true

# emit (name, detail, bubbles)

emit在实体上发出自定义DOM事件。例如，我们可以发出事件来触发动画：

// <a-entity animation="property: rotation; to: 0 360 0; startEvents: rotate">
entity.emit('rotate');

我们还可以将事件详细信息或数据作为第二个参数传递：

entity.emit('collide', { target: collidingEntity });

默认情况下，事件将冒泡。我们可以通过冒充泡沫来告诉它不要泡沫：

entity.emit('sink', null, false);

# flushToDOM (recursive)

flushToDOM将手动序列化实体组件的数据并更新DOM。阅读有关组件到DOM序列化的更多信息。

# getAttribute (componentName)

getAttribute 检索解析的组件数据（包括混合和默认值）。

// <a-entity geometry =“ primitive：box; width：3”>

entity.getAttribute（'geometry'）; 
// >> {原始：“盒子”，深度：2，高度：2，宽度：3，...}

entity.getAttribute（'geometry'）.primitive; 
// >>“盒子”

entity.getAttribute（'geometry'）.height; 
// >> 2

entity.getAttribute（'position'）; 
// >> {x：0，y：0，z：0}
如果componentName不是注册组件的名称，getAttribute 则其行为将与通常情况相同：

// <a-entity data-position =“ 0 1 1”>

entity.getAttribute（'data-position'）; 
// >>“ 0 1 1”

# getDOMAttribute (componentName)

getDOMAttribute仅检索在DOM中或通过明确定义的已解析组件数据setAttribute。如果componentName是已注册组件的名称，getDOMAttribute则将仅返回HTML中定义为解析对象的组件数据。getDOMAttribute对于组件来说，是部分形式，getAttribute因为返回的组件数据不包括应用的混合或默认值：

比较以上示例的输出getAttribute：

// <a-entity geometry =“ primitive：box; width：3”>

entity.getDOMAttribute（'geometry'）; 
// >> {原语：“盒子”，宽度：3}

entity.getDOMAttribute（'geometry'）.primitive; 
// >>“盒子”

entity.getDOMAttribute（'geometry'）.height; 
// >>未定义

entity.getDOMAttribute（'position'）; 
// >>未定义

# getObject3D (type)

getObject3D查找孩子THREE.Object3D的引用type上object3DMap。

AFRAME.registerComponent（'example-mesh'，{ 
  init：function（） { var el = this .el;     el.setObject3D（'mesh'，新的 THREE.Mesh（））;     el.getObject3D（'mesh'）;  //返回刚刚创建的THREE.Mesh。  } }）;

# pause ()

pause()将停止动画和组件定义的任何动态行为。当我们暂停实体时，它将停止其动画并调用 Component.pause()其每个组件。这些组件决定实施暂停时发生的情况，这通常会删除事件侦听器。pause()当我们暂停实体时，该实体将调用其子实体。

// <a-entity id =“ spinning-jumping-ball”>
 entity.pause（）;

例如，暂停时的外观控件组件将删除侦听输入的事件处理程序。

# play ()

play()将启动由动画和组件定义的任何动态行为。当DOM附加实体时，将自动调用此方法。当为实体时play()，该实体会调用play()其子实体。

entity.pause（）; 
entity.play（）;

例如，正在播放的声音组件将开始播放声音。

# setAttribute (componentName, value, [propertyValue | clobber])

如果componentName不是注册组件的名称，或者该组件是单属性组件，则setAttribute其行为将与通常情况相同：

entity.setAttribute（'visible'，false）; 尽管if componentName是已注册组件的名称，但它可能会对该值进行特殊解析。例如，位置组件是单属性组件，但是其属性类型解析器允许它获取一个对象：

entity.setAttribute（'position'，{ x：1，y：2，z：3 }）;
destroy ()

清理与实体相关的内存，例如清除所有组件及其数据。

# 更新多属性组件数据

要更新多属性组件的组件数据，我们可以将注册组件的名称作为componentName传递，并将属性的对象作为传递value。字符串也是可以接受的（例如type: spot; distance: 30），但是对象将为A-Frame节省一些解析工作：

//只有在对象中传递的属性将被覆盖。
entity.setAttribute（'light'，{ 
  type：'spot'，
  distance：30，
  强度：2.0
 }）;

或更新多属性组件的单个属性，我们可以将注册组件的componentName名称作为传递，将属性名称作为第二个参数传递，并将属性值设置为第三个参数：

//材质组件的所有先前属性（颜色除外）将不受影响。
entity.setAttribute（'material'，'color'，'crimson'）;

请注意，数组属性类型的行为是唯一的：

数组是可变的。它们是通过引用分配的，因此对组件的更改将对组件可见。 数组类型属性的更新不会触发组件的update方法，也不会发出事件。

# 放置多属性组件数据

如果true将用作第三个参数传递给.setAttribute，则未指定的属性将被重置并破坏：

//灯光组件的所有先前属性将被删除并覆盖。
entity.setAttribute（'light'，{ 
  类型：'spot'，
  距离：30，
  强度：2.0
 }，true）;

# setObject3D (type, obj)

setObject3D将注册过obj，一个THREE.Object3D作为type 下实体object3DMap。A-Frame添加obj为实体root的子代object3D。实体在调用时将发出object3dset带有type事件详细信息的事件setObject3D。

AFRAME.registerComponent（'example-orthogonal-camera'，{ 
  update：function（） { 此 .el.setObject3D（'camera'，新的 THREE.OrthogonalCamera（））;   } }）;

# removeAttribute (componentName, propertyName)

如果componentName是已注册组件的名称，以及从DOM中删除该属性，removeAttribute也会从实体分离该组件，并调用组件的remove生命周期方法。

entity.removeAttribute（'geometry'）;  //分离几何部件。
entity.removeAttribute（'sound'）;  //分离声音组件。
如果propertyName指定，removeAttribute则会将指定的那个属性的属性值重置propertyName为该属性的默认值：

entity.setAttribute（'material'，'color'，'blue'）;  //颜色为蓝色。
entity.removeAttribute（'material'，'color'）;  //将颜色重置为默认值白色。

# removeObject3D (type)

removeObject3Dtype从实体的 对象中移除指定的对象THREE.Group，从而从场景中移除。这将更新实体的 object3DMap值，将type密钥的值设置为null。通常从组件中调用，通常在remove处理程序中：

AFRAME.registerComponent（'example-light'，{ 
  update：function（） { this .el.setObject3D（'light'，new THREE.Light（））; //光线现在已成为场景的一部分。// object3DMap.light现在是THREE.Light（）对象。  }，
    

    


  delete：function（） { this .el.removeObject3D（'light'）; //现在从场景中移除了灯光。// object3DMap.light现在为null。  } }）;

# removeState (stateName)

removeState将从实体弹出状态。这将发出 stateremoved事件，我们可以使用.is以下命令检查其删除状态：

entity.addEventListener（'stateremoved'，function（evt） { if（evt.detail.state === 'selected'）{ console .log（'实体不再被选择。'）;   } }）;
  
    



entity.addState（'selected'）; 
entity.is（'selected'）;  // >>是

entity.removeState（'selected'）; 
entity.is（'selected'）;  // >>错误

# 侦听组件更改

我们可以使用该componentchanged事件来监听对实体的更改：

entity.addEventListener（'componentchanged'，function（evt） { if（evt.detail.name === 'position'）{ console .log（'实体已移至'，evt.target.getAttribute（'position'），'！'）;   } }）;

# 聆听附着和分离的子元素

当场景附加或分离实体时，我们可以使用child-attached和child-detached事件来监听：

entity.addEventListener（'child-attached'，function（evt） { 如果（evt.detail.el.tagName.toLowerCase（）=== 'a-box'）{ console .log（'已附加Box元素'）;   }; }）;
