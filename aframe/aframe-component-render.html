# 渲染器组件

该renderer系统配置了场景的 THREE.WebGLRenderer实例。输入WebVR / WebXR时，它还配置演示文稿属性。

# 例

<a-scene renderer="antialias: true;
                   colorManagement: true;
                   sortObjects: true;
                   physicallyCorrectLights: true;
                   maxCanvasWidth: 1920;
                   maxCanvasHeight: 1920;"></a-scene>

# 特性

属性 描述 默认值
抗锯齿 是否执行抗锯齿。如果为auto，则在移动设备上禁用了抗锯齿功能。 auto
颜色管理 是否使用颜色管理的线性工作流程。 false
高刷新率 在Oculus Browser上切换72hz模式。默认为60hz。 false
foveationLevel 在VR中启用偏爱功能以提高性能。0无，1低，2中，3高 0
sortObjects 是否在渲染之前对对象进行排序。 假
物理上正确的灯光 是否使用物理上正确的光衰减。 假
maxCanvasWidth 最大画布宽度。使用大小乘以设备像素比率。如果设置为-1，则不限制画布宽度。 1920年
maxCanvasHeight 最大画布高度。行为与maxCanvasWidth相同。 1920年
logarithmicDepthBuffer 是否使用对数深度缓冲区。 汽车
精确 片段着色器精度：低，中或高。 高
α 画布是否应包含Alpha缓冲区。 真正

注意：场景初始化后，这些属性可能不再更改。

# 抗锯齿

启用后，将以中等的性能成本平滑曲线和对角线上的锯齿状边缘。默认情况下，在移动设备上禁用抗锯齿。

# 颜色管理

色彩管理可提供更准确的渲染，并减少场景显得过亮或“褪色”的可能性。建议启用颜色管理以精确匹配纹理和建模工具中的颜色，但是此时非官方组件可能并不总是能够正确响应颜色管理。

在其他引擎和工具中，托管和非托管色彩模式分别类似于线性和伽玛工作流程。

注意：在three.js和A-Frame的早期版本中，提供了一个gammaOutput: true属性。启用颜色管理后，将自动应用此功能。

# 高刷新率

切换给定设备的最高刷新率。目前，Oculus Go中的Oculus浏览器支持此功能，并将渲染从60hz切换到72hz。

# foveationLevel

设置请求的偏心级别，在某些系统上处于立体渲染模式时，该级别将在视口边缘周围渲染较少的像素。Oculus Go上的Oculus浏览器当前支持此功能，其值范围从0（无）到3（高）。

# sortObjects

排序用于尝试正确渲染具有一定透明度的对象。由于各种限制，适当的透明度通常需要进行一些仔细的设置。默认情况下，不对对象进行排序，并且DOM中元素的顺序决定了呈现的顺序。对DOM元素重新排序提供了一种强制行为一致的方法，而使用DOM元素renderer="sortObjects: true"可能会导致摄像机移动时发生不必要的更改。

# 物理上正确的灯光

默认情况下，根据传统上常见但物理上不准确的模型，点光源和点光源会衰减（或随着它们变得更远而显得更暗）。要获得更真实的光衰减，请设置renderer="physicallyCorrectLights: true"。进行此更改时，可能需要调整光强度。两种模式下的性能均不会受到明显影响。

注意：如果glTF模型包含灯光，请使用物理上正确的灯光模式来匹配原始建模工具中的结果。

# logarithmicDepthBuffer

对数深度缓冲区可以在包含比例和距离差异很大的场景中提供更好的排序和渲染。

# 精确

在片段着色器中设置精度。主要用途是解决较旧的硬件/驱动程序中的问题。基于Adreno 300系列GPU的手机尤其成问题。您可以设置mediump为解决方法。它将改善性能，尤其是在移动设备中，但要注意，这可能会导致着色器/纹理中出现视觉伪像。

# α

画布是否应包含Alpha缓冲区。如果为真，则渲染器将具有透明的后缓冲区，并且画布可以与网页的其余部分合成。有关更多信息，请参见此处。
