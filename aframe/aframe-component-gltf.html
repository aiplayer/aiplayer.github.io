# gltf模型

glTF（GL传输格式）是Khronos的一个开放项目，为3D资产提供了一种通用且可扩展的格式，该格式既高效又可与现代Web技术高度互操作。

该gltf-model组件使用glTF（.gltf或.glb）文件加载3D模型。

请注意，glTF是一个相当新的规范，并且采用率还在不断增长。在three.js glTF加载程序上的工作和转换器仍处于活动状态。

注意： A-Frame支持glTF 2.0。对于使用旧版glTF格式的模型，请gltf-model-legacy从donmccurdy / aframe-extras使用。

# 为什么要使用glTF？

与仅支持顶点，法线，纹理坐标和基本材质的较旧OBJ格式相比，glTF提供了一组更强大的功能。除上述所有功能外，glTF还提供：

  • 层次对象
  • 场景信息（光源，摄像机）
  • 骨骼结构和动画
  • 更坚固的材质和着色器

对于没有动画的简单模型，OBJ仍然是常见且可靠的选择。

与COLLADA或FBX相比，支持的功能非常相似。但是，由于glTF专注于提供“传输格式”而不是编辑器格式，因此它与Web技术更具互操作性。以此类推，.PSD（Adobe Photoshop）格式对于编辑2D图像很有帮助，但是图像会转换为.JPG以在网络上使用。同样，glTF是传输3D资产同时呈现相同结果的一种更简单的方法。

简而言之，期望glTF模型与A-Frame一起使用比其他格式更可靠。

# 例

通过指向src为glTF文件指定的资产来加载glTF模型。

<a-scene>
  <a-assets>
    <a-asset-item id="tree" src="/path/to/tree.gltf"></a-asset-item>
  </a-assets>

  <a-entity gltf-model="#tree"></a-entity>
</a-scene>

# 值

类型 描述
selector 选择
string url()-glTF文件的封闭路径

# 事件

活动名称 描述
model-loaded glTF模型已加载到场景中。
model-error 无法加载glTF模型。

# 内联加载

或者，通过直接在中指定路径来加载glTF模型 url()。但是，场景在渲染之前不会等待资源加载。

<a-entity gltf-model="url(/path/to/tree.gltf)"></a-entity>

# 使用动画

如果要使用glTF模型中的动画，则可以使用aframe-extras中的animation-mixer组件。默认情况下，所有动画都循环播放。

<a-entity gltf-model="#monster" animation-mixer></a-entity>

# 使用Draco进行几何压缩

glTF模型中的几何可以使用Draco库进行压缩。对于主要包含几何图形，具有简单的无纹理材料或顶点颜色的模型，压缩通常可以将文件大小减少90–95％。当模型包含其他大数据（例如Draco不会影响的纹理或动画）时，文件大小的节省将变得不那么重要。

任何压缩形式的折衷都将是解码时间。压缩的模型需要较少的时间来下载和使用较少的带宽，但是只有在将其解压缩后才能呈现。为避免在VR中丢帧，请延迟体验的开始，直到下载并解压缩模型为止。

要将Draco压缩应用于现有的glTF模型，请使用 glTF-Pipeline。您还需要将Draco解码器库与场景一起托管，并按以下说明配置场景属性。

场景属性 当使用通过Draco压缩的glTF模型时，您必须在场景中托管Draco解码器库并配置解码器的路径：

<a-scene gltf-model="dracoDecoderPath: path/to/decoder/;">
  <a-entity gltf-model="url(pony.glb)"></a-entity>
</a-scene>
属性 描述 默认值
dracoDecoderPath Draco解码器库的路径。 ''

解码器文件夹必须包含三个文件：

  • draco_decoder.js — Emscripten编译的解码器，与任何现代浏览器兼容。
  • draco_decoder.wasm — WebAssembly解码器，与较新的浏览器和设备兼容。
  • draco_wasm_wrapper.js — WASM解码器的JavaScript包装器。

所有文件都可从three.js存储库中的 examples / js / libs / draco / gltf下获得。该gltf-model组件将自动选择使用WASM还是JavaScript解码器，因此两者都应包括在内。
