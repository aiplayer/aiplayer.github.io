# 可视化检查器和开发工具

本节将介绍许多有助于改善虚拟现实开发体验的有用工具：

  • A-Frame检查器-检查器工具，用于获取场景的不同视图并查看调整实体的视觉效果。虚拟现实模拟浏览器的DOM检查器。可以用++i在任何A帧场景中打开。

  • 键盘快捷键。

  • 运动捕捉-一个工具，记录和回放耳机和控制器的姿势和事件。点击录制，在虚拟现实耳机内移动，与控制器交互。然后在任何一台计算机上回放这些记录，以便快速开发和测试。减少进出耳机的时间。

我们将讨论构建在无需代码即可使用的A-Frame之上的GUI工具。以及其他可以简化跨多台机器开发的工具。

# A-Frame检查员

A帧检查器是用于检查和调整场景的可视化工具。有了检查员，我们可以：

  • 使用控制柄和辅助对象拖动、旋转和缩放实体

  • 使用小部件调整实体的组件及其属性

  • 立即查看更改值的结果，而不必在代码和浏览器之间来回切换

检查器类似于浏览器的DOM检查器，但为3D和A-Frame量身定制。我们可以切换检查器打开任何一个A帧场景在野外让我们查看源！

# 打开检查员

最简单的方法是按键盘上的快捷键。这将通过CDN获取检查器代码，并在检查器中打开我们的场景。同样的快捷键将关闭检查器。

我们不仅可以在检查器中打开本地场景，还可以使用检查器在野外打开任何A帧场景（只要作者没有显式禁用它）。

有关为检查器的本地、开发或自定义生成提供服务的详细信息，请参阅检查器自述文件。

# 使用检查员

# 场景图

检查员的场景图是基于树的场景表示。我们可以使用场景图来选择、搜索、删除、克隆和添加实体或导出HTML。

场景图列出的是A帧实体，而不是内部的3.js对象。给定HTML也是场景图的一种表示，检查器的场景图紧密地映射底层HTML。实体使用其HTML ID或HTML标记名显示。

# 视区

视口从检查器的角度显示场景。我们可以旋转、平移或缩放视口以更改场景的视图：

旋转：按住鼠标左键（或在轨迹板上向下一个手指）并拖动

平移：按住鼠标右键（或在轨迹板上向下两个手指）并拖动

缩放：上下滚动（或在触控板上用两个手指滚动）

从视口中，我们还可以选择实体并变换它们：

选择：在实体上单击鼠标左键，双击可将相机聚焦到该实体上

变换：在视口右上角选择一个辅助工具，拖动实体周围的红/蓝/绿辅助工具进行变换

# “组件”面板

“组件”面板显示选定实体的组件和属性。我们可以修改常用组件的值（例如位置、旋转、缩放），修改附加组件的值，添加和移除mixin，以及添加和移除组件。

每个属性的小部件类型取决于属性类型。例如，布尔使用复选框，数字使用值幻灯片，颜色使用颜色选择器。

我们可以复制各个组件的HTML输出。这对于可视化地调整和查找组件的所需值，然后将其同步回源代码非常有用。

# 运动捕捉

房间级的虚拟现实开发可能很麻烦。每次更改代码，我们都必须：

打开网页（通常在单独的计算机上运行）

进入虚拟现实

戴上耳机

抓住控制器（通常必须重新打开它们）

使用耳机和控制器进行测试

取下耳机和控制器，打开浏览器开发工具

如果有必要，请重新启动浏览器，因为它们当前处于实验状态并且有缺陷

重复

房间级虚拟现实开发变成了糖蜜。但是我们已经提出了一个工作流程来加速虚拟现实的开发，这样我们就可以自动化，快速开发，并使用a帧运动捕捉组件。

通过运动捕捉组件，我们可以在任何没有耳机的设备上记录虚拟现实动作（例如，耳机和控制器移动、控制器按钮按下），并重复播放这些虚拟现实动作。
