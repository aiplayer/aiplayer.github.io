# 链接

链接组件在体验之间进行连接，并允许在VR网页之间进行遍历。通过事件激活后，链接组件会将用户发送到其他页面，就像普通的网页重定向一样。要跨页面维护VR，必须满足以下条件：

浏览器是最新的WebVR规范，并实现了该vrdisplayactivate事件。 目标网页侦听window [ vrdisplayactivate]事件并进入VR。默认情况下，A-Frame体验的行为方式是这样的。 目前，in-VR导航仅由Firefox桌面，Oculus Browser，Samsung Internet和Supermedium实现。

# 链接用户体验

VR中的链接可以是任何东西，例如抓住物体，在我们的头上放一些东西，点击一些东西甚至吃东西！我们提供链接的默认实现，作为门户或窗口，使用360°图片缩略图预览目标。

随着我们在内容和浏览器中进行迭代和实验，链接遍历的UX将继续完善。

# 控制项

链接组件作为门户的默认UX可以使用游标或控制器进行交互。没有在A-Frame核心中直接提供此功能，但是A-Frame示例中有一个链接控制组件。该组件显示工具提示，并提供用于窥视和进入门户的按钮映射。

# 例

<a-scene>
  <a-assets>
    <img id="homeThumbnail" src="home.jpg">
  </a-assets>

  <a-entity link="href: index.html; title: My Homepage; image: #homeThumbnail"></a-entity>
</a-scene>

我们还提供了具有不同语法的链接原语：

<a-link href="index.html" title="My Homepage" image="#homeThumbnail"></a-link>

# 特性

属性 描述 默认值
backgroundColor 门户的内部（背景）颜色。 red
borderColor 门户的边框颜色。 white
hightlighted 布尔值可切换链接突出显示颜色。 false
HighlightColor 突出显示时的边框颜色。 '＃24CAFF'
href 链接指向的目标URL。 ''
image 在门户中将360°图像用作场景预览。可以是元素或URL 的选择器。 ''
on 要监听的事件将触发链接遍历。 "click"
peekMode 360°图像是否完全展开以进行预览。 false
title 链接上显示的文字。该href如果没有定义使用或页面URL。 ''
titleColor 链接上显示的文本的颜色。 white
visualAspectEnabled 是否启用门户的默认外观。如果我们要实现自己的链接遍历模式或形式，则设置为false。 true

# 手动导航

要手动导航，请使用window.locationAPI。请参阅有关navigation的StackOverflow问题。例如：

window.location.href = 'https://supermedium.com/supercraft/';
// or window.location.replace
