# 托管和发布

本节将介绍部署、托管和将A-Frame站点及其资产发布到Web上供全世界查看的几种方法。

# 发布网站

有许多免费服务可以部署和托管站点。我们将讨论一些更简单或流行的选项，但肯定还有其他选项，如AWS、Heroku或自托管。一个重要的注意事项是，由于浏览器的WebVR API的常见安全限制，这些站点应该使用SSL/HTTPS。下面的所有选项都适用于SSL/HTTPS。

# Glitch

“Glitch是一个友好的社区，在这里，你可以创建你梦想中的应用程序，使用示例应用程序进行重新混合、代码编辑器进行修改、即时托管和部署—任何人都可以免费在Glitch上构建web应用程序。”

故障是从浏览器中创建和发布网站最简单、最快的方法。小故障允许我们添加代码和文件、上传资产、与其他人一起编辑、定义自己的URL名称，并在每次更改时立即部署更改。我们甚至不需要注册或登录：

  • 试着去A-Frame Starter Glitch。

  • 点击Remix your own复制项目。

  • 单击左上角的“项目信息和选项”图标重命名我们的应用程序（例如，https://yoursitename.glitch.me）。

  • 编辑HTML，添加文件，修改项目。

  • 单击Show查看应用程序（例如，starter故障位于https://aframe.glitch.me）。

  • 对项目的每次更改都会立即更新应用程序。这可以通过登录、单击用户头像和在更改时切换刷新应用程序来关闭。

# Neocities

“创建自己的免费网站。无限的创造力，零广告。Neocities是一个由129100个网站组成的社交网络，这些网站正在找回失去的个人创造力。我们提供免费的网站托管和工具，让您创建自己的网站。加入我们！”

Neocities也是从浏览器中创建和发布网站的另一种免费而简单的方式。虽然它没有故障的一些特性，但Neocities很友好，允许我们将资产上传到项目目录，而不是CDN。这使得新兴城市至少在模型托管方面做得更好。有了Neocities，我们可以创建和编辑文件。然后将为我们主办和出版（例如，ngokvin.neocities.org）

# Surge

“面向前端开发人员的静态web发布简单、单命令的web发布。不离开命令行，免费发布HTML、CSS和JS。”

Surge是一个工具，可以通过命令行中的一个命令发布网站。如果您熟悉命令行，这是一个很好的工具。

# Github页面

“你和你的项目的网站。直接从GitHub存储库托管。只需编辑、推送，您所做的更改即会生效。”

如果项目在GitHub上，则GitHub页面是发布项目的首选方式。最简单的方法是转到项目的GitHub设置，向下滚动到GitHub页面部分，然后设置为发布master分支。这将在https://.github.io/上发布项目。

或者，如果不希望发布主更改，我们可以设置发布gh pages分支。命令行工具（如ghpages）可以使这成为一个命令发布过程。

# 托管Assets文件

我们还将讨论托管资产，如音频、纹理、模型和视频。 如果A-Frame站点与它的资产一起发布在同一个目录中（即，同一个域），那么我们不需要太担心托管资产。A-Frame站点可以使用相对url来引用资产，而且由于它们位于同一个域中，因此获取该资产没有问题。例如，如果您的所有资源都在同一个根目录中，并且我们通过Neocities、GitHub页面或spreag发布所有内容，那么就不会有问题。

一旦我们发布了我们的优秀项目，我们会想分享它，让其他人潜水！

# 创建媒体

A-Frame和VR非常直观；我们想为我们的项目制作视频和gif。

首先，我们要录制屏幕。在OS X上，使用内置QuickTime播放器的屏幕录制或OBS工作室录制屏幕。在Windows上，我们可以使用OBS Studio。OBS Studio还支持在屏幕上方流式播放和合成网络摄像头图像，这有助于在现实中（甚至在混合现实中）显示使用耳机的人。

然后，我们可能想修剪视频。在OS X上，我们可以使用QuickTime播放器的修剪工具（+t）。

要使用一个命令转换为GIF，请使用gifpardy。gifpardy在引擎盖下使用ffmpeg和gifsicle：

gifpardy in.mp4
gifpardy in.mp4 out.gif
gifpardy -r 320x240 --delay 8 in.mp4
