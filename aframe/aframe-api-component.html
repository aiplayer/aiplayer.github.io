# 组件

在实体组件系统模式中，组件是可重复使用的模块化数据块，我们将其插入实体中以添加外观，行为和/或功能。

在A框架中，组件会修改场景中作为3D对象的实体。我们将组件混合在一起并组成复杂的对象。他们让我们将three.js和JavaScript代码封装到可以从HTML声明式使用的模块中。

作为一个抽象的类比，如果我们将智能手机定义为实体，则可以使用组件赋予其外观（颜色，形状），定义其行为（调用时振动，低电量关闭）或添加功能（相机） ，屏幕）。

组件大致类似于CSS。就像CSS规则如何修改元素的外观一样，组件属性也可以修改实体的外观，行为和功能。

# 组件HTML表单

组件以一个或多个组件属性的形式保存一桶数据。组件使用此数据来修改实体。考虑一个发动机 组件，我们可以定义诸如马力或汽缸的属性。

HTML属性表示组件名称，这些属性的值表示组件数据。

# 单属性组件

如果组件是单属性组件，则意味着其数据由单个值组成，那么在HTML中，该组件值看起来像普通的HTML属性：

<！-`position`是位置组件的名称。-> 
<！-`1 2 3`是位置分量的数据。-> 
< a-entity  位置 = “ 1 2 3” > </ a-entity >

# 多功能组件

如果组件是多属性组件，则意味着数据由多个属性和值组成，那么在HTML中，组件值类似于内联CSS样式：

<！-`light`是灯光组件的名称。-> 
<！-灯光的“类型”属性设置为“点”。-> 
<！-灯光的“颜色”属性设置为“深红色”。-> 
< a-entity  light = “类型：点；颜色：深红色” > </ a-entity >

# AFRAME.registerComponent (name, definition)

注册一个A框架组件。我们必须先注册组件，然后才能在中的任何位置使用它们。根据HTML文件的含义，组件应排在之前。

  • {string} name-组件名称。通过HTML属性名称表示的组件的公共API。
  • {Object} definition-组件定义。包含架构和生命周期处理程序方法。
// Registering component in foo-component.js
AFRAME.registerComponent('foo', {
  schema: {},
  init: function () {},
  update: function () {},
  tick: function () {},
  remove: function () {},
  pause: function () {},
  play: function () {}
});

<!-- Usage of `foo` component. -->

    <script src="aframe.min.js"></script>
    <script src="foo-component.js"></script>
  </head>
  <body>
    <a-scene>
      <a-entity foo></a-entity>
    </a-scene>
  </body>
</html>

# 自定义属性类型

我们还可以通过提供parseand stringify 函数代替a 来定义自己的属性类型type：

schema: {
  // Parse slash-delimited string to an array (e.g., `foo="myProperty: a/b"` to `['a', 'b']`),
  // stringify array to string (e.g., `['a', 'b']` to `foo="myProperty: a/b"`)
  myProperty: {
    default: [],
    parse: function (value) {
      return value.split('/');
    },
    stringify: function (value) {
      return value.join('/');
    }
  }
}

parse通过setAttribute方法更新组件属性时调用。 stringify通过flushToDom方法更新DOM时调用。

# 单属性架构

组件可以是单属性组件（由一个匿名值组成）或多属性组件（由多个命名值组成）。A框架将根据架构的结构推断组件是单属性还是多属性。

单属性组件的架构包含type和/或default键，并且架构的值是纯值而不是对象：

AFRAME.registerComponent('foo', {
  schema: {type: 'int', default: 5}
});
<a-scene>
  <a-entity foo="20"></a-entity>
</a-scene>

.tick ()在场景的渲染循环的每个刻度或帧上调用。场景将调用组件的tick处理程序：

# dependencies

dependencies如果组件依赖一个或多个其他组件，则允许控制组件初始化的顺序。dependencies数组中指定的组件名称将在初始化当前组件之前从左到右初始化。如果依赖项具有其他依赖项组件，则这些其他依赖项组件将以相同的方式排序：

//最后初始化。
AFRAME.registerComponent（'a'，{ 
  依赖项：[ 'b' ] 
}）;

//初始化第二个。
AFRAME.registerComponent（'b'，{ 
  依赖项：[ 'c' ] 
}）;

//首先初始化。
AFRAME.registerComponent（'c'，{}）;

# multiple

该multiple标志允许组件在实体上具有其自身的多个实例。由于默认multiple设置为false，因此实体只能有一个组件的单个实例。例如，一个实体只能有一个几何组件。

但是，如果组件将multiple设置为true，则该组件可以具有多个实例：

AFRAME.registerComponent（'foo'，{ 
  multiple：true，// ... }）;

在DOM中，我们可以通过提供双下划线和ID（__）后缀来区分组件实例。例如，要附加声音组件的多个实例：

< a- 场景 > < a-entity 声音 = “ src：url（sound.mp3）” sound__beep = “ src：url（beep.mp3）” sound__boop = “ src：url（boop.mp3）”   > </ a-实体 > </ a-场景 >

从组件生命周期处理程序方法中，我们可以使用区分组件实例this.id。如果组件实例设置为 foo__bar，this.id则将为"bar"：

AFRAME.registerComponent（'foo'，{ 
  multiple：true，

  更新：功能（） { console .log（'此组件实例具有ID'，此 .id）;   } }）;

如果我们正在做a setObject3D()，我们通常会想使用this.attrName。如果组件实例设置为foo__bar，this.attrName则将为 foo__bar。这为我们提供了一个名称空间和一个ID，用于object3D在实体的上设置object3DMap：
