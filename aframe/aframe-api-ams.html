# 资产管理系统

A-Frame拥有资产管理系统，该系统使我们能够将资产放置在一个位置，并预加载和缓存资产以提高性能。请注意，资产管理系统仅用于预加载资产。在运行时在实体上设置的资产可以通过指向资产的直接URL来完成。

传统上，游戏和丰富的3D体验会在渲染场景之前预加载其资源，例如模型或纹理。这样可以确保资产在视觉上不会丢失，这对于确保场景在渲染时不尝试获取资产的性能很有帮助。

# 跨域资源共享（CORS）

由于A-Frame使用XHR获取资产，因此，浏览器安全性要求浏览器在跨域资源共享（CORS）标头的情况下为资产提供资产。否则，我们必须将资产托管在与场景相同的原始位置。

对于其他选项，GitHub Pages使用CORS标头提供一切。我们建议将GitHub Pages作为一个简单的部署平台。或提供设置了CORS标头的文件的另一项服务（例如Amazon S3）。

# 设置超时

我们可以设置一个超时时间，当到达时，场景将开始渲染，并且实体将开始初始化，而不管是否所有资产都已加载。默认超时为3秒。要设置不同的超时时间，我们只需将毫秒数传递给timeout属性：

如果某些资产的加载时间很长，我们可能需要设置适当的超时时间，以使用户在网络速度缓慢的情况下不必整天等待。

A-Frame使用这些进度事件，将浏览器缓冲的时间与资产的持续时间进行比较，以检测资产何时加载。

# 内部运作方式

从A字架继承的每个元素的AFRAME.ANode 原型。ANode控制加载和初始化顺序。对于一个元素进行初始化（不管它是，，，或 ），其子女必须已经初始化。节点自下而上初始化。

是一个ANode，它在加载之前会等待其子级加载。并且由于是的子项，因此场景必须有效地等待所有资产加载。我们还添加了额外的加载逻辑，以 使它们明确地等待加载（如果已定义）。 用于THREE.FileLoader提取文件。three.js将返回的数据存储在中THREE.Cache。每three.js所装载继承 THREE.FileLoader，无论是一ColladaLoader，OBJLoader， ImageLoader，等他们都可以访问和了解中央的 THREE.Cache。如果A-Frame已获取文件，则A-Frame不会尝试再次获取该文件。

因此，由于我们阻止了实体在资产上的初始化，因此在实体加载时，所有资产都已被获取。只要我们定义 s，并且实体使用某种形式获取文件 THREE.FileLoader，那么缓存将自动工作。
